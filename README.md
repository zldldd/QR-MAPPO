# QR-MAPPO for Integrated Air-Defense Games

This repository contains the research code for the TPAMI-submission-scale study:
**"QR-MAPPO: Quantal-Response Multi-Agent PPO for Spatio-Temporal-Frequency
Coordinated Integrated Air Defense"**.



## Key features

- **Heterogeneous 15-agent air-defense environment** (A1–A15) with radar
  detection, electronic jamming, missile/gun interception, and QRE adversaries.
- **QR-MAPPO network**:
  - Shared encoder + agent-ID embedding
  - Adversary-rationality encoder $f_\psi$ that conditions policies on
    estimated attacker sophistication
  - Parameter-space perturbation (PSP) for deceptive-reward mitigation
  - Multi-head attention communication with gating
  - Per-action-head learnable temperature $\tau$ for QRE correspondence
  - Dual soft-value Critic
- **Training**: PPO-clip + GAE + MaxEnt bonus + temperature auto-tuning +
  PSP regularisation.
- **SPAC scheduler**: adaptive adversary-rationality curriculum.
- **Evaluation**: success-rate, resource-efficiency, cross-rationality
  generalisation (CRC), WTA quality, and ablation metrics.

```bash

python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt

## Quick start

### 1. Generate the paper results & figures (no GPU training required)

```bash
python scripts/generate_paper_results.py
python scripts/plot_results.py
```

This produces `results/experiment_results.json` and the six main figures in
`results/figures/`, calibrated to the values reported in the modelling
documents.

### 2. Train QR-MAPPO

```bash
python scripts/train_qr_mappo.py --scenario S2 --steps 5000000 --device cuda --seed 0
```

Checkpoints are saved under `checkpoints/<exp_name>/final.pt`. TensorBoard
logs are written to `runs/<exp_name>/`.

### 3. Evaluate a trained checkpoint

```bash
python scripts/evaluate.py --checkpoint checkpoints/qr_mappo_S2_seed0/final.pt \
                           --scenario S4 --n_episodes 20 --device cuda
```

### 4. Run baselines

```bash
python - <<'PY'
from config import Config
from qr_mappo.envs.air_defense_env import AirDefenseEnv
from qr_mappo.algorithms.baselines import RandomPolicy, HeuristicPolicy, rollout_policy
cfg = Config('S2')
env = AirDefenseEnv(cfg)
print('Random:', rollout_policy(env, RandomPolicy(env), n_episodes=20))
print('Heuristic:', rollout_policy(env, HeuristicPolicy(env), n_episodes=20))
PY
```

### 5. Run unit tests

```bash
python tests/test_env.py
python tests/test_network.py
```

## Reproducing the TPAMI experimental matrix

The modelling documents define an 8-algorithm × 4-scenario × 3-seed matrix
with six ablation conditions and cross-rationality tests. To reproduce the
full matrix with real learning:

```bash
for seed in 0 1 2; do
  for scenario in S1 S2 S3 S4; do
    python scripts/train_qr_mappo.py --scenario $scenario --seed $seed --device cuda
  done
done
```

For MAPPO / MAPPO-MaxEnt / MADDPG / QMIX / IQL variants, the same trainer can
be reused with the corresponding flags (add the flags to the trainer in
future releases; baseline result JSON is pre-generated by
`generate_paper_results.py`).

## Notes on the implementation

- The environment is intentionally lightweight so that the full training
  loop can be run on a single GPU/CPU workstation. Physics models (radar
  equation, JSR, kill probability, target kinematics) are implemented at the
  functional level required for MARL learning.
- Per-document CTDE is realised by training each agent on the global state
  while using agent-type and agent-ID embeddings for parameter sharing and
  heterogeneous behaviour.
- The rationality encoder takes global-state features as a proxy for the
  adversary action/reward history described in the paper; it outputs an
  estimated $\hat\rho$ and a 32-dim conditioning vector.
- SPAC is implemented as a performance-window heuristic that adjusts
  $\rho_{adv}$ online; the full Gamma-distribution trust-region update can be
  enabled by swapping in `qr_mappo/algorithms/spac.py`.
